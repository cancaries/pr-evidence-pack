pr-evidence-pack

Turn a branch diff into scope, tests, manual checks, risk, rollback, and reviewer focus before a maintainer has to ask.

View on GitHub Use the CLI
Zero dependencies No network calls Plain Markdown output

Built for the review gap

Summaries are common. Evidence is still missing.

For contributors

Package the context reviewers usually need: what changed, why the scope is tight, what was tested, and how to roll it back.

For maintainers

Review starts from concrete evidence instead of a confident but thin PR description.

For agent-assisted work

Make AI-assisted changes easier to inspect without adding another bot to the repository.

Run locally

A CLI command that makes review evidence explicit.

git clone https://github.com/cancaries/pr-evidence-pack.git
cd pr-evidence-pack
node ./bin/pr-evidence-pack.js \
  --base main \
  --issue "#123" \
  --scope "CLI evidence generation" \
  --out-of-scope "Hosted review bot behavior" \
  --test "node --test" \
  --manual "Checked generated Markdown locally" \
  --risk "Low: Markdown-only output" \
  --rollback "Revert this PR"

Current install path

Generated output

Markdown that a reviewer can actually use.

Scope Evidence Risk Rollback Reviewer focus 
# PR Evidence Pack

## PR Snapshot
- Branch: feature/evidence-output
- Base: main
- Working tree: clean

## Scope
### In Scope
- Add first-class scope and out-of-scope evidence fields
- Classify repository metadata files as config

### Out of Scope
- Executing user supplied commands
- Posting comments to remote forges

## Evidence
- Linked issue or ticket: local product-quality hardening
- Tests run: node --test
- Manual verification: Reviewed generated Markdown locally

## Risk and Rollback
- Risk: Low: output structure and warning logic only
- Rollback: Revert the PR

## Missing Evidence
- No missing evidence detected.

Workflow

Use it before a PR exists.

01

Compare the branch

Read local git metadata, changed files, commits, and diff stats.

02

Add human evidence

Record the tests, manual checks, risk, rollback, and explicit non-goals.

03

Submit with context

Paste the evidence pack or PR body draft where reviewers will see it.

Safety boundary

Narrow by design.

It does

  • Read git metadata locally.
  • Generate Markdown evidence.
  • Warn when review evidence is missing.
  • Work before any hosted PR exists.

It does not

  • Run user-supplied commands.
  • Send repository data to a service.
  • Post comments automatically.
  • Claim that a PR is correct.
